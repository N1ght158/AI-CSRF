# 命令使用

## Python 方式

```bash
python src/csrf_autopilot.py run \
  --frontend https://github.com/acme/web \
  --backend https://github.com/acme/api \
  --provider github \
  --base main \
  --auto-merge off \
  --workspace .
```

执行仓库准备（会做 clone/fetch 和建分支）：

```bash
python src/csrf_autopilot.py run \
  --frontend https://github.com/acme/web \
  --backend https://github.com/acme/api \
  --provider github \
  --base main \
  --workspace . \
  --branch-prefix ai/csrf-fix \
  --execute-bootstrap
```

## PowerShell 方式

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\csrf-autopilot.ps1 `
  -Frontend https://github.com/acme/web `
  -Backend https://github.com/acme/api `
  -Provider github `
  -Base main `
  -AutoMerge off `
  -Workspace .
```

执行仓库准备：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\csrf-autopilot.ps1 `
  -Frontend https://github.com/acme/web `
  -Backend https://github.com/acme/api `
  -Provider github `
  -Base main `
  -Workspace . `
  -BranchPrefix ai/csrf-fix `
  -ExecuteBootstrap
```

## 输出结果

命令执行后会在 `reports/` 下生成两份文件：

- `run-plan-<run_id>.json`
- `run-plan-<run_id>.md`

当前行为为 dry-run，仅输出计划，不会修改目标仓库代码。

补充说明：

- 默认是 dry-run，不会访问远端仓库。
- 追加 `--execute-bootstrap`（或 `-ExecuteBootstrap`）后，会执行远端访问检查、clone/fetch 与工作分支创建。
- 追加 `--require-token`（或 `-RequireToken`）后，如果未检测到令牌变量会直接失败。
