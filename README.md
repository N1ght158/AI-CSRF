# AI-CSRF

注：本人练手项目。

这是一个用于自动化修复 CSRF 风险的项目，目标是做一个面向前后端仓库的 AI CSRF 自动修复插件：自动分析、修复、测试、提交关联 PR，并在检查通过且满足策略时自动合并。

## 目标

1. 接收前后端仓库地址。
2. 分析 CSRF 防护缺口。
3. 生成修复改动与测试。
4. 自动提交 PR，并在满足策略时合并。

项目文档见 `docs/` 目录。

## 目前进展

- 目标 1：已完成。支持接收前后端仓库地址、地址校验、run-id 生成，并已支持仓库准备流程，包括远端访问检查、`clone/fetch` 和工作分支创建。
- 目标 2：初步完成。当前已完成静态规则扫描，可以生成 CSRF 风险识别报告，检查内容包括后端 CSRF token、Origin/Referer 校验、Cookie 安全属性、疑似关闭 CSRF、防护 header 注入、token 来源和凭据配置等；同时已加入修复决策层，可以基于扫描结果生成修复决策报告、优先级、目标阶段和 AI 执行提示。简单点说，就是已经能初步识别 CSRF 缺口并生成修复决策。
- 目标 3：初步完成。当前已完成后端修复 MVP，支持在显式传入 `--apply-backend-fix` 时，对 Express CommonJS 后端项目生成 CSRF middleware、接入入口文件，并生成最小测试；前端修复和更多后端技术栈还未接入。
- 目标 4：未完成。自动提交 PR 和自动合并还未接入，目前只有流程参数框架。

## 说明

当前阶段默认不会直接修改目标仓库代码，主要是先跑流程。只有显式传入 `--apply-backend-fix`，并且识别到支持的后端项目结构时，才会在本地后端工作分支生成修复改动。
